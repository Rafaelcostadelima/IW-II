Atividade de IW-II

O HTML é como se fosse a planta de uma casa: tem paredes, portas, móveis, etc. O problema, é que tudo é estático, nada se mexe. A função do DOM (Document Object Model) é de dar vida a essa planta. Com o DOM, o navegador lê o HTML como uma árvore: <body> com um galho, <h1> como um ramo e <button> como uma folha.

JavaScript usa o DOM como um controle remoto. Com ele, o JS pode mandar que o DOM entenda quando que botão for clicado para mudar de cor de fundo, por exemplo.

Tecnicamente todos os elementos de um site (até o que a gente não vê) são nós (node).

  • Elementos são aquelas divisões: <div>, <section>, <body> etc. Eles servem como divisões (caixas) que organizam o conteúdo. É neles que você geralmente clica ou aplica estilos.

  • Atributos são as informações extras que dicam dentro da tag de abertura de um elemento, como o scr ou class ou href etc.

  • Textos já são o conteúdo real, as palavras que aparecem na tela. Tipo o "Clique aqui!" na tag "<button>Clique aqui!</button>"

Um exemplo que resumo esse contexto de elementos, atributos e textos de um nó utilizando o Document Object Model:

No código <a href='https://github.com/'>Visite aqui!</a> :

  • Nó do elemento: <a>
  • Nó de atributo: href='https://github.com/'
  • Nó de texto: "Visite aqui!"

Há três metódos que são como ferramentas de busca que o JavaScript usa para encontrar os nós dentro da "árvore" do DOM. Cada um tem a sua especialidade:

  • "getElementById()": Ele busca pelo atributo de ID que se dá a um elemento no HTML. Ele retorna o único elemento que possui aquele o ID (caso não haja, null).

  • "querySelector()": é o método mais moderno e versátil, pois usa a mesma sintaxe dee seletores do CSS. O desenvolvedor pode utilizar tag, classe, ID ou qualquer combinação (ex.: .botao-azul , #topo etc.). Apenas o primeiro elemento queele encontrar que combine com a busca.

  • "querySelectorAll()" : Funciona de forma parecida com o querySelector, porém, ele não para no primeiro resultado que consegue, ele verifica todos os elementos que batem com o seletor CSS que foi indicado para ele e os retorna em formato de uma NodeList (simplesmente só uma lista de elementos). Para poder acessar cada um, geralmente precisa de um laço de repetição com o .forEach().

Há dois elementos que podem ser confundidos no DOM: innerHTML e innerText.

A principal diferença entre esses elementos está em como o JavaScript trata o que está dentro da tag: um vê o código e o outro vê apenas o texto puro.

  • "innerHTML": Ele interpreta as strings como HTML real. Se você colocar tags dentro da string, o navegador vai renderizá-las (criar novos elementos). Ele lê ou aktera todo o conteúdo, incluindo as tags HTML internas.

  • "innerText": Ele trata tudo como texto simples. Mesmo que você escreva tags HTML, elas aparecerão escritas literalmente na tela, como se fossem parte de uma frase. Lê ou altera apenas o texto visível que o usuário final enxerga

Os eventos funcionam com o "sentido de tato" (interatividade) da página web. Eles são sinais enviados pelo navegador para o seu código JavaScript avisando que algo aconteceu na interface (como um clique, um movimento do mouse etc.). Sem eventos, os sites seriam apenas um cartaz estático, sem vida, interatividade e tudo mais. Com eles, se torna um software que reage ao usuário, dando vida e interatividade.

  • "onclick": ocorre quando o usuário clica em um elemento (botão, link, imagem, etc.). É o evento mais comum no mundo web. Normalmente é usado para enviar um formulário, abrir um menu ou mudar a cr de um fundo, etc.

  • "onchange": ocorre quando o valor de um elemento de formulário é alterado e o usuário "sai" dele (perde o foco). É usado normalmente para validar se um e-mail foi digitado corretamente ou atualizar o preço total quando alguém muda a quantidade de um ponto num carrinho.

  • "onmouseover": ocorre no momento em que o ponteiro do mouse entra na área de um elemento. Por exemplo: mostrar um balão de ajuda (tooltip), destacar um card ou mudar a cor de um link para dar feedback visual
